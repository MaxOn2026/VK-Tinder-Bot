# VK Tinder Bot 🎯

Бот для знакомств ВКонтакте с функционалом Tinder. Ищет подходящих кандидатов по параметрам, позволяет ставить лайки, создавать взаимные мэтчи и общаться в чате.

## 🚀 Возможности

- 🔍 **Умный поиск** — находит кандидатов по возрасту, городу, интересам
- ❤️ **Лайки и мэтчи** — взаимные симпатии создают мэтчи
- 💬 **Чат с мэтчами** — общение с теми, кто лайкнул в ответ
- 📊 **Статистика** — просмотрено анкет, лайков, мэтчей
- ⚙️ **Настройки** — гибкие параметры поиска
- 🚫 **Чёрный список** — блокировка нежелательных анкет
- 🔐 **Админ-панель** — рассылка, статистика, управление
- 📸 **Фотографии** — показывает топ-3 фото кандидатов

## 🛠 Технологии

- **Python 3.14**
- **SQLAlchemy** — ORM для работы с БД
- **PostgreSQL** — реляционная база данных
- **VK API** — интеграция с ВКонтакте
- **vk_api** — библиотека для работы с VK API
- **Black** — автоматическое форматирование кода

## 📖 Описание

VK Tinder Bot — это полнофункциональный бот для знакомств, который работает через ВКонтакте. Бот находит подходящих кандидатов на основе предпочтений пользователя, позволяет лайкать, блокировать и просматривать анкеты. При взаимном лайке создаётся мэтч, после чего пользователи могут общаться через встроенный чат. Все данные хранятся в надёжной PostgreSQL базе данных.

### 🎯 Основные возможности

✅ **Умный поиск** — находит кандидатов по возрасту, полу, городу  
✅ **Система лайков** — добавление в избранное и чёрный список  
✅ **Топ-3 фото** — показывает лучшие фото кандидатов по лайкам  
✅ **Навигация** — кнопки "Следующий", "Предыдущий"  
✅ **Списки** — просмотр избранных и заблокированных  
✅ **Управление** — удаление из списков  
✅ **🎉 Мэтчи** — автоматическое определение взаимных лайков  
✅ **💬 Чат** — общение между пользователями с мэтчем  
✅ **⚙️ Настройки** — изменение возраста и расстояния поиска  
✅ **📊 Статистика** — просмотры, лайки, блокировки, мэтчи  
✅ **🔐 Админ-панель** — управление пользователями и рассылка  
✅ **📢 Рассылка** — массовые сообщения всем пользователям  
✅ **PostgreSQL** — надёжное хранение данных  
✅ **SQLAlchemy ORM** — профессиональная работа с БД  
✅ **Потокобезопасность** — scoped_session для многопоточности  
✅ **Пул соединений** — эффективное управление подключениями

## 🚀 Установка

### 1. Клонируйте репозиторий

```bash
git clone https://github.com/ваш-username/VK-Tinder-Bot.git
cd VK-Tinder-Bot
```

### 2. Создайте виртуальное окружение

**Windows:**
```powershell
python -m venv venv
venv\Scripts\activate
```

**Linux/Mac:**
```bash
python3 -m venv venv
source venv/bin/activate
```

### 3. Установите зависимости

```bash
pip install -r requirements.txt
```

### 4. Настройте PostgreSQL

Подключитесь к PostgreSQL и выполните:

```sql
-- Создайте базу данных
CREATE DATABASE vkinder_db;

-- Подключитесь к базе
\c vkinder_db;

-- Установите расширение для поиска по триграммам
CREATE EXTENSION IF NOT EXISTS pg_trgm;
```

### 5. Настройте переменные окружения

Создайте файл `.env` в корне проекта:

```env
# VK API токены
GROUP_TOKEN=vk1.a.ВАШ_ТОКЕН_ГРУППЫ
USER_TOKEN=vk1.a.ВАШ_ПОЛЬЗОВАТЕЛЬСКИЙ_ТОКЕН
GROUP_ID=ID_ВАШЕЙ_ГРУППЫ_БЕЗ_МИНУСА

# PostgreSQL
DB_HOST=localhost
DB_PORT=5432
DB_NAME=vkinder_db
DB_USER=postgres
DB_PASSWORD=ваш_пароль
```

**Как получить токены:**

- **GROUP_TOKEN**: Создайте группу ВК → Управление → Работа с API → Создать ключ
- **USER_TOKEN**: Перейдите на [vkhost.github.io](https://vkhost.github.io/), выберите Kate Mobile, авторизуйтесь и скопируйте токен из URL
- **GROUP_ID**: Откройте группу, ID — это число после `club` в URL (например, `club123456` → `123456`)

### 6. Инициализируйте базу данных

```bash
python init_db.py
```

Должно появиться:
```
🔧 Инициализация базы данных...
✅ Подключение к БД установлено
✅ Таблицы созданы
✅ База данных готова к работе!
```

## 💬 Чат между мэтчами

Когда два пользователя лайкают друг друга, создаётся **мэтч**. После этого они могут общаться через встроенный чат.

### Как использовать чат:

1. **Посмотрите список мэтчей:**
   ```
   🎉 мэтчи
   ```
   Бот покажет список с номерами:
   ```
   🎉 Ваши мэтчи:
   
   1. Иван Иванов (ID: 123456)
      💬 Напишите 'чат 1' чтобы начать общение
   
   2. Пётр Петров (ID: 789012)
      💬 Напишите 'чат 2' чтобы начать общение
   ```

2. **Начните чат:**
   ```
   чат 1
   ```
   Бот ответит:
   ```
   💬 Чат с Иван Иванов
   
   Напишите сообщение или 'выйти' для выхода из чата
   ```

3. **Отправьте сообщение:**
   Просто напишите текст — он отправится вашему мэтчу:
   ```
   Привет! Как дела?
   ```
   Бот подтвердит:
   ```
   ✅ Сообщение отправлено Иван Иванов
   ```

4. **Выйдите из чата:**
   ```
   выйти
   ```

### Особенности чата:

- ✅ Сообщения доставляются через VK API
- ✅ Можно общаться с несколькими мэтчами одновременно
- ✅ История сохраняется в БД (таблица `messages`)
- ✅ Уведомления о новых сообщениях
- ✅ Команда `выйти` для выхода из режима чата

### Команды чата:

| Команда | Описание |
|---------|----------|
| `🎉 мэтчи` | Показать список всех мэтчей |
| `чат N` | Начать чат с N-м мэтчем |
| `выйти` / `выход` | Выйти из режима чата |

## 🔧 API DatabaseManager

### Класс `DatabaseManager`

Управляет подключениями к PostgreSQL через SQLAlchemy.

#### Основные возможности

- **Пул соединений**: QueuePool (10 основных, 20 дополнительных)
- **Проверка соединений**: `pool_pre_ping=True`
- **Потокобезопасность**: `scoped_session`
- **Контекстный менеджер**: автоматический commit/rollback
- **Сырой SQL**: выполнение произвольных запросов

#### Методы

##### `initialize(db_url: Optional[str] = None) -> None`

Инициализирует подключение к БД.

```python
from database.db_manager import db_manager

# Из config.py
db_manager.initialize()

# Или явно
db_manager.initialize("postgresql://user:pass@host:5432/dbname")
```

##### `create_tables(drop_first: bool = False) -> None`

Создаёт все таблицы.

```python
db_manager.create_tables(drop_first=False)
```

##### `get_session() -> Generator[Session, None, None]`

Контекстный менеджер для сессии.

```python
with db_manager.get_session() as session:
    users = session.query(BotUser).all()
    # Автоматический commit или rollback
```

##### `execute_raw(query: str, params: dict = None) -> Any`

Выполняет сырой SQL.

```python
results = db_manager.execute_raw(
    "SELECT * FROM bot_users WHERE city = %(city)s",
    {"city": "Москва"}
)
```

##### `close() -> None`

Закрывает все соединения.

```python
db_manager.close()
```

## 🔨 Разработка

### Рекомендации

1. **Всегда используйте контекстные менеджеры** для работы с сессиями:

```python
with db_manager.get_session() as session:
    # Операции с БД
    pass
```

2. **Обрабатывайте транзакции** для сложных операций:

```python
try:
    with db_manager.get_session() as session:
        # Несколько операций
        session.commit()
except Exception as e:
    session.rollback()
    raise
```

3. **Избегайте N+1 запросов**, используя `lazy="selectin"` или `lazy="joined"`

4. **Закрывайте соединения** при завершении работы:

```python
db_manager.close()
```

### Обработка ошибок

| Ошибка | Причина | Решение |
|--------|---------|---------|
| `psycopg2.OperationalError` | Нет подключения к БД | Проверьте параметры в `.env` |
| `SQLAlchemyError` | Ошибка запроса | Используйте try/except с rollback |
| `IntegrityError` | Нарушение уникальности | Проверьте существование записи |
