# VKinder Bot

**VKinder Bot** — это Telegram-бот для социальной сети ВКонтакте, который помогает пользователям находить потенциальных партнёров через автоматизированный поиск по заданным критериям. Бот анализирует профили пользователей ВКонтакте, находит подходящие варианты на основе возраста, пола и города, а также предоставляет возможность взаимодействия: добавление в избранные, просмотр фото, чёрный список и система взаимных мэтчей.

Проект реализован на Python с использованием SQLAlchemy ORM для работы с базой данных PostgreSQL и Bot LongPoll API ВКонтакте для получения сообщений, а также User API для поиска людей.

## Обзор

**VKinder Bot** — это Telegram-бот для социальной сети ВКонтакте, который помогает пользователям находить потенциальных партнёров через автоматизированный поиск по заданным критериям. Бот анализирует профили пользователей ВКонтакте, находит подходящие варианты на основе возраста, пола и города, а также предоставляет возможность взаимодействия: добавление в избранные, просмотр фото, чёрный список и система взаимных мэтчей.

Проект реализован на Python с использованием SQLAlchemy ORM для работы с базой данных PostgreSQL и Bot LongPoll API ВКонтакте для получения сообщений, а также User API для поиска людей.

### Ключевые технологии

- **Python 3.8+** — язык программирования
- **SQLAlchemy 2.0** — ORM для PostgreSQL (использует `Mapped` и `mapped_column`)
- **vk_api** — клиент API ВКонтакте (Bot LongPoll + User API)
- **PostgreSQL 12+** — база данных
- **Bot LongPoll API** — реальное время обработки сообщений

## Возможности

- **Автоматизированный поиск партнёров** — поиск потенциальных партнёров по настраиваемым критериям (возраст, пол, город) через VK API `users.search`
- **Галерея фотографий** — просмотр фотографий профилей с сортировкой по популярности (по лайкам)
- **Интерактивная клавиатура** — навигационные кнопки для переключения между кандидатами
- **Управление избранным** — сохранение понравившихся профилей в список избранных
- **Чёрный список** — блокировка нежелательных профилей
- **Система взаимных мэтчей** — распознавание взаимных симпатий и создание мэтчей
- **Исключение дубликатов** — автоматическая фильтрация просмотренных, избранных и заблокированных пользователей
- **Управление состоянием пользователей** — отслеживание навигации и предпочтений пользователей

## Установка

### Требования

- Python 3.8+
- PostgreSQL 12+
- Токен группы ВКонтакте (для отправки сообщений)
- Токен пользователя ВКонтакте (для поиска людей)
- Группа ВКонтакте (для работы бота через Bot LongPoll)

### Шаг 1: Клонирование репозитория

```bash
git clone https://github.com/ваш-аккаунт/VK-Tinder-Bot.git
cd VK-Tinder-Bot
```

### Шаг 2: Создание виртуального окружения

```bash
python -m venv .venv
source .venv/bin/activate  # или .venv\Scripts\activate на Windows
```

### Шаг 3: Установка зависимостей

```bash
pip install -r requirements.txt
```

### Шаг 4: Настройка переменных окружения

Создайте файл `.env` в корне проекта:

```env
# Настройки PostgreSQL
HOST=localhost
PORT=5432
DATABASE=vk_tinder
DB_USER=postgres
PASSWORD=ваш_пароль

# Настройки ВКонтакте
VK_GROUP_TOKEN=ваш_токен_группы
VK_USER_TOKEN=ваш_токен_пользователя
VK_API_VERSION=5.199

# Дополнительные настройки (опционально)
DB_ECHO=false
DB_ECHO_POOL=false
```

#### Получение токенов ВКонтакте

1. **Токен группы**: Создайте группу ВКонтакте и сгенерируйте токен с правами на `messages`, `users` и `photos`

2. **Токен пользователя**: Используйте скрипт `get_token.py` для получения токена пользователя через авторизацию в браузере:

```bash
python get_token.py
```

Это откроет окно браузера для авторизации. Скопируйте токен из терминала и добавьте его в файл `.env`.

### Шаг 5: Создание базы данных

База данных будет создана автоматически при первом запуске бота, или вы можете создать её вручную:

```bash
psql -U postgres -c "CREATE DATABASE vk_tinder;"
```

### Основные команды в чате

Бот использует текстовые команды без слэшей. Также доступны кнопки клавиатуры.

| Команда | Описание |
|---------|----------|
| `начать`, `привет`, `/start`, `hi`, `hello` | Начало работы с ботом |
| `начать поиск` | Начать поиск новых пользователей |
| `следующий`, `следующий >>` | Показать следующего пользователя |
| `предыдущий`, `<< предыдущий` | Показать предыдущего пользователя |
| `в избранное`, `❤️ в избранное` | Добавить текущий профиль в избранное |
| `в чёрный список`, `🚫 в чёрный список` | Пропустить текущего пользователя |
| `🏠 главное меню`, `главное меню` | Вернуться в главное меню |
| `список партнёров` | Показать список партнёров |
| `список избранных` | Показать список избранных |
| `матчи` | Показать список матчей |
| `чёрный список` | Показать чёрный список |
| `❌ удалить из избранного` | Удалить текущего пользователя из избранного |
| `✅ удалить из чёрного списка` | Удалить текущего пользователя из чёрного списка |

### Главное меню

При первом запуске бот предлагает начать поиск. После этого в главном меню доступны:

- **Начать поиск** — начать поиск новых партнёров
- **Список партнёров** — просмотреть текущий список партнёров
- **Список избранных** — просмотреть избранных пользователей
- **Матчи** — просмотреть пользователей с взаимным лайком
- **Чёрный список** — просмотреть заблокированных пользователей

## API документация

### Модуль базы данных (`database/`)

#### DatabaseManager

Управляет подключениями к PostgreSQL с использованием SQLAlchemy и пула соединений.

```python
from database.db_manager import db_manager

# Инициализация подключения к БД
db_manager.initialize()  # Использует настройки из config.py
# или
db_manager.initialize("postgresql://user:pass@host:5432/dbname")

# Создание таблиц
db_manager.create_tables()

# Использование сессии с контекстным менеджером
with db_manager.get_session() as session:
    users = session.execute(select(BotUser)).scalars().all()

# Выполнение сырого SQL
results = db_manager.execute_raw("SELECT * FROM users WHERE city = %(city)s", {"city": "Москва"})

# Закрытие подключений
db_manager.close()
```

**Методы:**

- `initialize(db_url: Optional[str] = None) -> None` — Инициализирует подключение к базе данных
- `create_tables(drop_first: bool = False) -> None` — Создаёт все таблицы базы данных
- `get_session() -> Generator[Session, None, None]` — Получает сессию базы данных
- `execute_raw(query: str, params: dict | None = None) -> Any` — Выполняет сырой SQL
- `close() -> None` — Закрывает все подключения к базе данных

## Разработка

### Структура кода

Проект организован в виде модульных компонентов:

| Модуль | Назначение |
|--------|------------|
| `main.py` | Точка входа в приложение |
| `bot_logic/bot.py` | Основной цикл бота и обработка Bot LongPoll |
| `bot_logic/handlers.py` | Обработчики сообщений и событий |
| `bot_logic/keyboards.py` | Клавиатуры ВКонтакте (JSON) |
| `bot_logic/vk_client.py` | Клиент для API ВКонтакте (два токена) |
| `bot_logic/vk_search.py` | Логика поиска и фильтрации пользователей |
| `bot_logic/state_manager.py` | Управление состоянием пользователей |
| `database/` | Операции с базой данных |
| `config.py` | Конфигурация приложения |

### Добавление новых функций

1. Реализуйте логику в соответствующем модуле `bot_logic/`
2. Добавьте обработчик в `bot_logic/handlers.py`
3. Добавьте команду в словарь `COMMANDS` в `bot_logic/bot.py`
4. Обновите клавиатуры в `bot_logic/keyboards.py`, если меняется UI
5. Обновите документацию
6. Тщательно протестируйте

### Миграции базы данных

Используйте Alembic для миграций схемы базы данных:

```bash
pip install alembic
alembic init migrations
```

Настройте `alembic.ini` и `migrations/env.py` для использования `db_manager.engine`.
